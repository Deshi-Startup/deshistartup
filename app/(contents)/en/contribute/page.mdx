# How to Contribute

Deshi Startup is an open, source-backed founder manual for Bangladesh. The goal is practical knowledge that people can improve, without letting unsourced or risky information slip through.

## Quick Contributions

Use **Edit this page** at the top of any page. You do not need a GitHub account.

The page text becomes editable in your browser. Click the sentence, heading, list item, or table cell you want to improve, type the correction, then choose **Review edits**. You will see the before/after text before submitting.

You can submit:

- a visible page edit for typos, clearer wording, missing citations, or a specific correction
- a serious issue report when something could mislead founders
- a source-backed rewrite suggestion when a larger improvement is needed

The edit becomes a pending revision in the public review queue. It does not change the live page until an editor approves it.

A useful pending edit usually includes:

- the page and section that should change
- the current sentence, paragraph, or link so editors can see a clear diff
- the proposed correction
- a deep link to an official or reliable source when possible
- why the issue is risky if it could mislead founders

## Serious Issues

Use **Report serious issue** for possible misinformation in legal, tax, VAT, RJSC, payment, employment, healthtech, privacy, import/export, or government incentive content.

These reports enter the review queue with `urgent`, `risk: high`, and usually `needs-expert-review` labels. A human editor or domain expert should check the issue before the published page changes.

## Review Queue

The public form creates a GitHub issue behind the scenes with the page path, review tier, source fields, and either a focused diff or a full-source draft diff. Contributors do not need to see GitHub, but maintainers still get a transparent audit trail.

Low-risk typo, grammar, and broken-link edits can be fast-tracked by one editor. Protected pages and high-risk factual edits need a citation and expert or senior editor approval before publishing.

## Trusted Editors

Trusted editors can use the `/admin/` dashboard to edit pages in the browser. The dashboard writes to GitHub, so every change keeps a public history.

Editor responsibilities:

1. triage pending public revisions
2. avoid publishing factual claims without sources
3. keep `risk_level: high`, `last_verified`, and a sources section on high-risk pages
4. request expert review when needed
5. wait for content QA and build checks before merge

## Expert Review

If you work in law, tax, banking, HR, healthtech, data privacy, export/import, fundraising, or market research, you can review specialist pages.

An expert review should state:

- which claims you checked
- which sources you used
- where there are caveats or process variations
- where founders should seek professional advice

## What We Do Not Publish

- unsourced legal, tax, or regulatory claims
- personal attacks, rumors, or unverified company/person claims
- information where AI is the only source
- promotional or affiliate content that does not neutrally help founders
