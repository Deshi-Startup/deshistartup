# কীভাবে অবদান রাখবেন

> **সারকথা:** এই সাইটের প্রতিটি পাতা যে কেউ উন্নত করতে পারেন – ঠিক উইকিপিডিয়ার মতো। ভুল ধরিয়ে দিতে ২ মিনিট লাগে, ছোট সংশোধনে ৫ মিনিট, আর একটি নতুন গাইড লিখতে এক সন্ধ্যা। ভাষা বা বানান নিখুঁত হতে হবে না – রিভিউয়াররা গুছিয়ে দেবেন।

বাংলাদেশে নিয়ম বদলায়, ফি বদলায়, নতুন প্রশ্ন আসে। এত বড় জ্ঞানভাণ্ডার অল্প কয়েকজন মিলে নির্ভুল ও হালনাগাদ রাখা অসম্ভব – দরকার অনেক চোখ আর অনেক হাত। আপনি যা জানেন (একটা লাইসেন্স করার অভিজ্ঞতা, একটা ব্যাংকের নিয়ম, একটা ভুল বানান), সেটাই কারও পরের ধাপ।

## পথ ১ – ভুল বা পুরোনো তথ্য জানান (২ মিনিট)

কোনো পাতায় ভুল, পুরোনো ফি, নষ্ট লিংক বা অস্পষ্ট ব্যাখ্যা চোখে পড়লে:

1. পাতার ওপরে বা নিচে **“ভুল জানান”** লিংকে ক্লিক করুন।
2. GitHub-এ একটি ফর্ম খুলবে – পাতার নাম আগে থেকেই লেখা থাকবে।
3. কী ভুল আর সঠিক তথ্যটা কী (জানলে সূত্রসহ) লিখে **Submit new issue** চাপুন।

GitHub অ্যাকাউন্ট না থাকলে বিনামূল্যে খুলে নিন – ইমেল ঠিকানা থাকলেই হয়।

## পথ ২ – ছোট সংশোধন নিজেই করুন (৫–১০ মিনিট)

বানান, লিংক, একটি বাক্য বা একটি সংখ্যা ঠিক করতে:

1. যে পাতাটি ঠিক করতে চান, সেখানে **“সম্পাদনা”** লিংকে ক্লিক করুন।
2. GitHub পাতাটির লেখা (Markdown ফরম্যাটে) খুলে দেবে। ডান দিকের পেনসিল আইকনে ক্লিক করুন – GitHub স্বয়ংক্রিয়ভাবে আপনার নামে একটি কপি (fork) বানিয়ে নেবে।
3. লেখাটি ঠিক করুন। ফরম্যাট নিয়ে ভয় পাবেন না – সাধারণ লেখা লিখলেই চলে।
4. নিচে **Propose changes** চেপে তারপর **Create pull request** দিন।
5. রিভিউয়াররা দেখে অনুমোদন করলে পরিবর্তনটি সাইটে চলে আসবে।

## পথ ৩ – নতুন গাইড লিখুন (সবচেয়ে বড় উপকার)

সাইটে শত শত বিষয় **“লেখা বাকি”** অবস্থায় আছে – প্রতিটি বিভাগের পাতার নিচে তালিকা দেখতে পাবেন। যে বিষয়টি আপনি জানেন:

1. বিষয়টির পাতায় ঢুকে **“পাতাটি লিখুন”** বোতামে ক্লিক করুন।
2. পুরোনো লেখা মুছে নিজের লেখা বসান। কাঠামো হিসেবে এই ছাঁচটি ব্যবহার করতে পারেন:

```text
# শিরোনাম

> **সারকথা:** ৩-৪ লাইনে পুরো বিষয়ের নির্যাস।

## কার কখন দরকার
## ধাপে ধাপে কী করবেন
## খরচ ও সময় (সাল উল্লেখ করুন)
## সাধারণ ভুল
## সূত্র (সরকারি লিংক আগে)
```

3. **Propose changes → Create pull request** দিন।

নিজের অভিজ্ঞতা থেকে লিখুন – “আমি ২০২৬-এ চট্টগ্রাম সিটি কর্পোরেশনে ট্রেড লাইসেন্স করেছি, ধাপগুলো এই ছিল” ধরনের বাস্তব লেখা সবচেয়ে মূল্যবান।

## লেখার নিয়মকানুন (সংক্ষেপে)

| নিয়ম | উদাহরণ |
|---|---|
| সহজ, প্রচলিত বাংলা; “আপনি” সম্বোধন | “আপনি আবেদন করবেন” – “আবেদন করা হইবে” নয় |
| প্রচলিত ইংরেজি টার্ম বাংলা হরফে রাখুন | ট্রেড লাইসেন্স, ভ্যাট, ব্যাংক অ্যাকাউন্ট, মার্কেটিং |
| নতুন টার্মের প্রথম ব্যবহারে ব্যাখ্যা দিন | “ইকুইটি (equity) মানে কোম্পানির মালিকানার ভাগ” |
| বাংলা বাক্যে বাংলা সংখ্যা | “ফি ৫০০ টাকা”, “ধাপ ৩” |
| আইন, ফি ও নিয়মের দাবিতে সূত্র দিন | সরকারি পোর্টাল (RJSC, NBR, বাংলাদেশ ব্যাংক) সবচেয়ে ভালো |
| ফি/তারিখ লিখলে সাল উল্লেখ করুন | “২০২৬ সালের হিসাবে ফি ৩,০০০ টাকা” |
| যা নিশ্চিত নন, লিখবেন না | অনুমান লিখলে “যাচাই প্রয়োজন” বলে দিন |

## রিভিউ কীভাবে হয়

প্রতিটি পরিবর্তন সাইটে যুক্ত হওয়ার আগে একজন রিভিউয়ার দেখেন: তথ্য কি সূত্রের সঙ্গে মেলে, ভাষা কি সহজ, কারও ক্ষতির ঝুঁকি আছে কি না। আপনার লেখা প্রথমবারে নিখুঁত হতে হবে না – **অসম্পূর্ণ কিন্তু সঠিক লেখা, না-লেখা পাতার চেয়ে সবসময় ভালো।**

## GitHub একেবারেই না পারলে

যেভাবে স্বাচ্ছন্দ্য, সেভাবেই লিখুন – [GitHub-এ একটি issue খুলে](https://github.com/Deshi-Startup/deshistartup/issues/new) লেখাটি সরাসরি পেস্ট করে দিলেও চলবে। কেউ একজন সেটি ঠিক জায়গায় বসিয়ে দেবেন।
